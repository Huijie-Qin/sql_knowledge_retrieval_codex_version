# SQL数据源解析工程

## 项目介绍
本工程解析 `案例/` 下的 `.sql` 和 `.md` 源文件，直接结合 Prompt 输入 LLM 进行端到端抽取，并按数据源生成标准化文档。

## 核心策略
1. 单源文件（<=1.5K行）采用全量输入解析，减少规则预提取导致的信息损失。
2. 数据源更新采用章节级分块合并 + token预算控制，降低上下文窗口爆炸风险。
3. 解析进度、案例用表索引、质量检察闭环全部落盘，支持断点续传和审计。

## 输出文件
- `数据源/<业务域>/<表名>.md`
- `数据源/解析进度.md`
- `数据源/案例表使用索引.json`
- `数据源/检察报告.md`

## 运行
```bash
python -m src.main
```
